Model & System Information

Why RandomForest?

  • Handles tabular financial/HR features effectively.
  • Robust against overfitting through ensembling.
  • Supports feature importance for interpretability.

Why SMOTE?

  • Balances underrepresented hiring states.
  • Improves classifier fairness across classes.

Risk Score Logic

Risk score is derived from model probability of the Slowdown class and scaled to 0–100.

0–35: Low, 35–65: Moderate, 65–100: High.

System Architecture

User Input → Flask API /predict → RandomForest Model → Prediction + Risk Score → Frontend Insights
